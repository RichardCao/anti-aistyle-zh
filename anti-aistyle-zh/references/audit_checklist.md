# 改后审计清单

改写完成后，按这张卡回扫。

## 快速路由

- 默认成稿不需要外显本清单；二改、验证、回归或用户要求审计时读取。
- 通用解释、总结、骨架问题查第 1-7 节。
- 小说、提纲、人物稿专项查第 8-10 节。
- 事实保真、体裁边界、检测器化和越界问题查第 11 节。
- 长文、多文件或第二轮改写才使用第 12 节记分卡。
- 本清单只判“是否完成”；具体改法回到 `rewrite_principles.md`、`longform_validation_rules.md`、`novel_longform_rules.md` 或 `plot_outline_rules.md`。

## 1. 解释与总结卡

检查：

- 有没有“也就是说 / 换句话说 / 可以看到 / 不难发现”在替读者解释
- 有没有连续 `3` 句以上仍在 `命名现象 -> 解释原因 -> 翻译意义`
- 段尾有没有再下定义、再盖章、再拔高
- 半数以上段落是否仍在用经验句、价值句、定义句收尾

## 2. 对比与经验句卡

检查：

- “不是……而是……”“问题不在……而在……”是否连续出现
- 这些对比句是在做辨异 / 认知推进，还是在替作者归纳
- 有没有“人最难的是……”“最后还是得……”这类老师腔经验句
- 把这些句子删掉后，事实和动作是否已经够了
- 如果只是换一批词，仍在反复回收同一个判断，也算残留

## 3. 汇报骨架卡

检查：

- 有没有 `首先 / 其次 / 更重要的是 / 最终需要` 在撑结构
- 有没有整篇像整理好的方法材料
- 有没有每段都像在交付阶段性结论
- 有没有“再往下看 / 更现实的一点 / 更麻烦的是 / 当然，也不是没有别的路径”这类软骨架过渡
- 如果原文领路词不高，改后却明显升高，直接判为 `转场回潮残留`

## 4. 护栏与限定卡

检查：

- 有没有“当然这不是说…… / 不能简单地…… / 也不意味着……”过密
- bare `当然` 作为段首让步句，是否比原文更高频
- 删掉这些限定后，原意是否仍成立
- 如果原文护栏句不高，改后却明显升高，直接判为 `安全限定回潮残留`

## 4.1 虚词与填缝卡

检查：

- `其实 / 已经 / 仍然 / 于是 / 同时 / 进一步 / 往往 / 反而` 这类虚词是否在改后更密
- 它们是不是主要长在段首、转句位、收束位，用来补平滑度而不是承担真实语义
- 同类虚词是不是连续出现，导致几句都像同一种“稳句”
- 原文没有这么多填缝词，改后却明显升高，直接判为 `虚词 / 关联词回潮残留`
- 判断时先和原文比，再和同体裁真人常态比；不是越少越好，而是不要靠这类词把稿子缝成统一腔

## 4.2 否定骨架卡

检查：

- `不是……而是…… / 问题不在……而在…… / 不只是……更…… / 不是没有……只是……` 是否连续出现
- 这些句式是在做必要辨异，还是只是在替作者把意义整理得更工整
- 同类句式是不是在段首、收束位和关键判断句上重复承担同一功能
- 原文否定-肯定骨架不高，改后却明显升高，直接判为 `否定骨架回潮残留`
- 不是完全禁用；只要频率高到像模板骨架，就说明还没完成

## 5. 重复功能卡

检查：

- 一件事有没有被证明两次以上
- 一个意思有没有举了三组以上同类例子
- 一段情绪有没有被抬了两次以上
- 长文里有没有不同段落在重复做同一种论证
- 连续 `3` 段以上是否仍在重复 `判断 -> 解释 -> 例子 -> 收束`
- 不同段落是不是只是换措辞、换例子，却仍在复写同一个 proposition

## 6. 抽象与机制卡

检查：

- `落地 / 路径 / 语义 / 边界 / 维度 / 颗粒度` 这类词是否过密
- 有没有“背景条件 / 默认选项 / 风险判断 / 成功想象”这类软抽象堆得太顺
- 一句话里是否连续堆了 `3` 个以上机制名词来承担解释

## 6.1 正式壳 / 非人称化卡

检查：

- 文本是不是明明在写人、关系、行动和后果，却被改成了状态、机制、条件和安排
- 有没有“相关工作 / 具体事项 / 有关方面 / 外部环境 / 当前阶段”这类非人称壳话在接管表达
- 该出现主体、对象、责任或压力的地方，是否被改成无人称、无对象的稳妥说明
- 如果原文对象明确，改后却更像正式说明或成熟材料，直接判为 `正式壳残留`
- 通知、材料说明、执行问答类文本，改后是否平白长出礼貌抬头、背景铺垫和发布口吻，而实际使用信息反而后移
- 原文已经正式，改后是否又被抬成更法言法语、更像制度释义或示范文本的版本

## 6.1.1 强体裁 / 卡片化卡

检查：

- 编号摘录、条文说明、执行问答、操作摘要是否被修成完整制度解读稿或培训稿
- 通知、FAQ、知识库、客服口径、内部周报、使用说明是否被压成等长短条，导致上下文变薄、关键词复用变密
- 原文是口播 / 社媒脚本时，是否被洗成 `小标题 + 命令句 + 禁止句` 的说明卡
- 评审意见回复、修改说明、审稿回复是否每条套同一组 `已补充 / 已调整 / 本研究 / 后续研究` 骨架
- 能直接落动作、条件、范围、后果、修改位置或不采纳原因的地方，是否又加了一层主持句、`关于` 段题、否定式解释或展望
- 如果删掉这层标题、虚词、否定解释或模板回复不损操作边界，判为 `强体裁壳残留`

## 6.2 认识过程 / 立场推进卡

检查：

- 结论是不是总比认识过程先到，像作者已经想明白了再回来写
- 有没有连续出现“先判断 -> 再解释 -> 最后回收立场”的稳定链条
- 本来应该保留的迟疑、误判、偏见修正、局部摇摆，是否被整理成了平滑认知路径
- 如果原文带摸索感，改后却只剩成熟判断，直接判为 `认识过程残留`

## 7. 长文结构卡

检查：

- 连续 `400-600` 字有没有不落回物件、动作、身体阻力、生活场景
- 评论 / 方法文是否仍在循环 `判断 -> 解释 -> 例子 / 方法 -> 收束`
- 随笔 / 叙事是否仍在循环 `现场 -> 回望 -> 领悟`
- 段群之间是不是总按同一种因果 / 转折 / 回收关系推进
- 能不能用一句话写出本文主导路线；如果能轻易写成标准模板，通常还太整
- 诊断阶段点过的骨架位，改后是否真的断开
- 删掉解释框后，是否又长出 `领路词 -> 判断 -> 护栏句`、等权分层或 `综合判断 -> 导向句 -> 高位结论` 的新稳骨架
- 评论包 / 组稿中的单篇是否被修成独立闭环专栏，完整得过了头

## 7.1 句法、断句与标点卡

检查：

- 长短句是不是波动太小，像同一种句型在反复上色
- 插入语、并列结构、转折结构是不是长期落在同一位置
- 逗号、分号、冒号、括注、引号是不是只在承担一种节奏功能
- 段首词、转句动作、收束手法是不是重复抬头
- 如果只是词更自然了，但断句和标点节奏几乎没变，仍算 `句法 / 断句 / 标点残留`

## 7.2 口播 / 审稿回复卡

检查：

- 原文不是口播稿，却被改出“像在对听众讲解”的节奏
- 有没有“你会发现 / 说白了 / 其实这里关键是 / 放到现实里看”这类讲述者牵手动作
- 书面文、叙事文、评论文是否被修成可直接讲出来的顺口话术
- 原文已经是口播稿时，是否被反向洗成 FAQ、通知或使用说明卡
- 评审意见回复是否每条都套同一回复骨架，而不是落到具体改动、数据边界或不采纳原因
- 命中前两类时分别判为 `口播腔迁移 / 口播卡片化残留` 或 `回复模板化残留`

## 8. 小说卡

检查：

- 小说章节里是不是反复出现“动作 -> 判断 -> 收束”的三连稳句
- 同一种情绪落锤模板是不是在多个章节重复承担同一功能
- 线索、证人、物件、反转是不是在按功能排队
- 显性解释删掉后，是不是换成了更软的抒情翻译句
- 短 scene 是否被洗成更亮、更短、更干净的微型成稿
- 相邻章节、主要人物或任务 scene 是否仍共享同一套发亮机制、微反应或分工模板

## 8.1 视角进入卡

检查：

- scene 或叙事段是不是只剩信息和动作，没有明确的感知入口
- 人物已经在场，但读者进场时摸不到他此刻先看到、先听到、先误认到什么，也摸不到代词、时空或注意力的落点
- 改写是不是只把信息讲清楚了，却没有保住沉浸的进入点
- 如果原文有进入口，改后却被抹平成俯视说明，直接判为 `视角进入残留`

## 8.2 多声部 / 对白区分卡

检查：

- 不同人物、不同身份、不同关系位置的说话声部是否被修成同一腔
- 对白是不是只剩信息推进，没有阶层、年龄、职业、心气、地域或代际差异
- 旁白、引语、对白、通知口吻之间的边界是否被抹平
- 如果主要人物的对白互换后也大体成立，直接判为 `多声部残留`

## 9. 提纲与剧情卡

检查：

- 每卷是不是都过于明确地承担一组功能任务
- 角色成长是不是被平均排班
- 真相推进是不是卷卷准时抬一格
- 中段几卷是不是在重复同一种揭示机制
- 是否连续多卷都靠旧物、残字、簿页、器物露底
- 是否缺少不升级、只下挖的一卷，或缺少白跑、误判、无功能场和低功率缓冲段
- 提纲正文是不是仍像规划壳话，而不是去壳后的结构文本
- 是否还保留运营壳、统一章节卡，或被铺成流畅卷介绍 / 项目导览 / 大段摘要
- 规划应有的块感、密度差、工作性和未抹平接口是否还在

## 10. 姿态卡

检查：

- 这段是在发生，还是在复盘
- 作者有没有跳出来总领全文
- 读者是不是被照顾得太明白
- 文本是不是过度清楚、过度体面、过度会控场
- 结尾是不是太会收
- 情绪是不是总被补一句“我终于明白了什么”
- 人物动作和表情之后，作者有没有又补一句明确心理
- 整篇是不是一直只有一种成熟、稳定、会收束的气口
- 作者是不是长期站在高位归纳、安顿、下结论

## 10.1 引语与对象感卡

检查：

- 引语、对话、通知口吻、访谈答话是不是被洗成和正文同一种气口
- 有对象的文本里，对象感、关系位置、现实压力以及地域 / 社群 / 职业声纹是否被抹平
- 讲话、公文、纪实、访谈、小说对白里，不同发话者是否被修成同一把声音
- 如果引语只剩信息，没有身份、场域和口气差异，也算残留

## 10.2 人物稿成稿化卡

检查：

- 人物稿是不是被修成“出场肖像 -> 几件代表性小事 -> 品性 / 手艺 / 活路收束”的标准人物成稿
- 人物是不是越来越像某种议题、职业伦理或生活态度的承载器
- 结尾是不是稳稳落在“体面 / 活路 / 手艺没断 / 城市边角的价值”这类副刊式盖章
- 如果黑词不多，但整篇越来越像成熟人物小稿，也判为残留

如果只是笼统觉得“太稳”，却找不到重复结构，先停手。

## 11. 保真与越界卡

检查：

- 原意有没有被改掉
- 重点有没有漂移
- 原文必要的判断有没有被过度削弱
- 口吻支点有没有丢失
- 事实类文本是否新增原文没有的事实、数据、关系、动机、职业身份、时间地点、因果链或专业结论
- 是否改了设定或名词体系
- 是否偷换叙述视角
- 是否为了去味伤到口吻支点
- 是否把演讲稿、公文、政策说明、学术摘要误洗成普通散文或口语评论
- 是否为了降低 AI 感人为打乱、过度碎片化、硬加错漏、口头禅、情绪噪音或无支撑细节

事实越界直接回写；检测器化只能说明改写变成另一种伪装，不算完成。

## 12. 极简记分卡

长文、多文件或第二轮改写时，至少补这张极简卡：

- `解释残留`：`0-5`
- `骨架 / 收束残留`：`0-5`
- `句法 / 标点残留`：`0-5`
- `结构 / 规划感残留`：`0-5`
- `姿态 / 人格残留`：`0-5`
- `伪具体残留`：`0-5`
- `压缩 / 摘要化残留`：`0-5`
- `转场 / 安全限定 / 虚词回潮`：`0-5`
- `否定骨架 / 正式壳残留`：`0-5`
- `视角进入 / 多声部残留`：`0-5`

每项至少给 `1` 处证据，不写空泛判断。

再补两条硬判断：

- 如果改后 `显性转场词`、`安全限定句`、同类填缝虚词或否定-肯定骨架的存在感高于原文，且主要承担缝合 / 归纳功能，不算本轮完成
- 如果长文明显跌破对应长度警戒线，又说不出具体删掉了哪几段重复功能，先判为 `压缩 / 摘要化残留`
