# 长文专项验证规则

这份文件只补三类专项：长文长度强审线、体裁边界、长文混合体的回潮与压平问题。核心判断不是“有没有黑词”，而是段落在做什么、连续几段是否在做同一件事、作者是否一直站在高位整理答案，以及改后会不会缩成提纯版标准答案。

这里所有“频率 / 密度 / 存在感”判断都先和原文比，再和同体裁、同长度、同运行方式的真人常态比；不是越少越好，而是不要高到像统一模板。

## 快速路由

- `3000+` 长文、强体裁、压缩风险、正式壳回潮时读本文件。
- 普通长文骨架查第 1-5 节。
- 演讲、公文、政策分析、诗歌、法规说明、公告、访谈等强体裁查第 6-7 节。
- 改后复核只查第 8 节，不要整包加载。
- 小说章节转到 [novel_longform_rules.md](./novel_longform_rules.md)。
- 提纲和卷表转到 [plot_outline_rules.md](./plot_outline_rules.md)。

## 1. 长文骨架卡

优先检查这些模式：

- 连续 `3` 句以上都在 `命名现象 -> 解释原因 -> 翻译意义`
- 超过半数段落都在用经验句、价值句、定义句收尾
- 连续 `3` 段以上都在重复 `判断 -> 解释 -> 例子 -> 收束`
- 段群之间总在同一种因果 / 转折 / 回收关系里推进
- 不同段只是换词重复同一个判断或同一个立场回收
- 一句话里连续堆 `3` 个以上机制名词
- 连续 `400-600` 字不落回物件、动作、身体阻力、生活场景
- 全文越写越顺，越顺越像标准件

执行卡：

- 删掉最后一层翻译
- 压掉同功能解释句
- 合并同功能段
- 打断至少一处过稳的段群关系，不让段与段总按同一种关系接力
- 删掉其中一段的收束句
- 让部分段落只停在事实、动作、物件或对话上
- 如果全文超过 `3000` 字，至少点出 `2` 处必须打断的骨架位
- 评论 / 政策评论若尾部重新长成 `综合判断 -> 导向句 -> 高位结论块`，默认未完成
- 如果正文明显在按“第一层 / 第二层 / 第三层”稳步铺开，改后仍保持等权层面推进，也默认未完成
- 如果全文已经从解释腔退下来，却变成“成熟评论员把问题讲透”的稳定成稿，同样按未完成处理

## 2. 人格与姿态卡

高风险信号：

- 作者长期站在高位归纳、安顿、下判断
- 情绪刚冒头，就在一两句内被整理成成熟判断
- 改后不像讲解稿了，却更像成熟评论员、顾问或主持人在控场

执行卡：

- 少留“我已经想明白了”的句子
- 留一点未讲圆的地方
- 让动作、对话、场景承担一部分理解工作
- 不要每次情绪出现都立刻翻译成体会

## 2.1 正式壳 / 非人称化卡

高风险信号：

- 人、对象、责任、压力被改成状态、条件、机制和安排
- 文本越来越像“有关方面 / 相关问题 / 当前阶段 / 具体事项”的说明壳
- 该落在具体人和具体后果上的句子，被改成无人称、无对象的成熟表述

执行卡：

- 能写人就不写“情况”，能写动作就不写“推进”
- 对象明确的文体，先保对象和后果，再谈句子整洁
- 不要用正式壳替代解释腔；那只是换了一层更体面的 AI 壳

## 2.2 认识过程卡

高风险信号：

- 结论总比观察先到，读者只能接收作者整理好的答案
- 本该保留的迟疑、误判、修正、分叉，被抹平成平顺认知链
- 全文几乎没有“怎么看出来”的过程，只有“已经想明白”的回收

执行卡：

- 保一点判断形成的过程，不要只留最后立场
- 允许局部先停在观察、摩擦、误会和未讲圆的感受上
- 不要把每一段都修成先见之明的成熟总结

## 3. 伪具体、句法与断句同色卡

高风险信号：

- 物件、地点、动作看起来变多了，但只是在给原命题垫脚
- 段落分工没变，作者姿态没变
- 连续 `3` 句以上都在用同一种稳句或收句方式
- 长短句分布、插入语位置、并列结构和断句手法几乎同色
- 逗号、分号、冒号、括注、引号等标点长期只承担一种节奏功能
- 改后词更自然了，但句长、转句动作、收束手法仍几乎同色

执行卡：

- 不只问“有没有具体介质”，还要问“介质有没有改变段落功能”
- 如果细节只是安全补丁，就继续往下拆骨架
- 至少拆异一半最显眼的同色句群
- 一处停在动作，一处停在物件，一处停在半句未收住的话
- 不要只换词；要同时打散句长、断句和标点承担的节奏功能

## 4. 转场、护栏与填缝回潮卡

高风险信号：

- 删掉解释框后，改稿反而补进更多显性转场词
- 解释腔下降了，但护栏句、让步句、稳妥限定变多了
- `首先 / 其次 / 更重要的是` 变少了，但段首 `当然` 变多了
- `其实 / 已经 / 仍然 / 于是 / 同时 / 进一步` 这类填缝词接管了连贯功能

执行卡：

- 不要为了“更顺”而把段与段缝得更明显
- 宁可让段落靠事实或观察自然转，不要回补领路词
- 如果一句话不需要提前防误解，就不要主动加限定
- 原文没有那么多显性领路词、护栏句和填缝词，改后也不要新添一层“稳定主持人”口气

## 4.1 过度清楚与过度体面卡

高风险信号：

- 逻辑被照顾得太明白，读者几乎没有自己完成理解的空间
- 改后更正式、更体面、更会控场，却没有更像人在写
- 文本不像解释稿了，却更像已经准备刊发的成熟成稿

执行卡：

- 少做高位总领和意义翻译
- 允许判断停在局部，不急着把所有关系讲成完整答案
- 不要把“更清楚”“更完整”“更体面”误当成“更自然”

## 5. 压缩回潮卡

高风险信号：

- 删掉解释框后，长文被直接压成更短、更整、更容易复述的提纯版
- 段落数没少太多，但每段都成了“主张句 + 一层补充”的轻量答案块
- 字数明显下跌，却说不出删掉的是哪些重复功能、护栏段、双面铺垫或空转解释
- 被删掉的主要是节奏误差、现场停顿、对象感和局部摩擦，留下来的却是判断骨架
- 检测分下降主要靠篇幅骤减换来

执行卡：

- 长文去味追求的是功能保真，不是机械等长
- 删掉重复、解释、护栏、双面铺垫后自然变短，属于正常结果
- 如果篇幅明显下跌，先检查是不是把多个段落功能压成一个结论段
- 优先恢复被压掉的局部观察、现场段、执行摩擦和人物反应，不要用更整的总结句补偿
- 命中长度线时，必须指出被压掉的是观察、阻力、对象感、段群还是现场

## 5.1 引语与对象感卡

高风险信号：

- 引语、对话、引用说明和正文用同一种成熟解释者气口
- 讲话、通知、访谈、纪实里，对象感和场域感被洗成通用自然散文
- 别人的话只剩信息，不再带场面压力、身份位置和口气差异

执行卡：

- 引语先保对象和口气，不要先保整洁
- 不要把不同人的话都修成同一把声音
- 对象感本来重要的文体，先保对象、用途、关系和压力，再谈句子顺滑

## 5.1.1 人物稿成稿化卡

高风险信号：

- 人物稿越来越像可直接刊发的副刊人物小稿
- 开头是完整人物介绍，中段是几件代表性小事，结尾再落到人物品性、手艺、体面或活路
- 黑词不多，但整篇过稳、过顺、过会收

执行卡：

- 不要把街头人物、工作人物、纪实人物修成统一人物模板
- 少做人物价值盖章，多留收摊、回家、下一单、手上余动作和没说完的话
- 如果 detector 主要对人物稿转正，先查是不是成稿化，而不是先查黑词

## 5.2 口播腔迁移卡

高风险信号：

- 原文不是口播稿，却被修出主持、讲解、直播式节奏
- 解释句下降后，换成“说白了 / 你会发现 / 放到现实里看”式领说
- 文字更顺口了，但更像要拿去讲，而不是拿来读

执行卡：

- 去味不是把书面文改成好讲的口语
- 需要保书面张力的文体，宁可略有棱角，也不要统一成顺口讲述腔
- 如果读起来像成熟主播、主持人或顾问在带听众走，就继续回写

长度警戒线：

- 评论 / 方法文：原文约 `80%`
- 人物特写 / 纪实非虚构：原文约 `75%`
- 演讲 / 政策分析：原文约 `80%`
- 诗歌 / 组诗：原文约 `70%`

## 6. 强体裁边界卡

### 演讲 / 政策分析 / 公文

高风险信号：

- 被洗成普通自然散文
- 对象感、正式层级、现实压力、执行口径被抹平
- 同时出现 `稳妥解释者回潮 + 压缩回潮`
- 讲话仍像完整发言稿模板：开场问候、三点展开、价值回收、感谢收束都过于准点
- 政策评论虽然保住了题材，但变成“成熟政策评论员”标准件，段与段过匀，争议点不够具体

执行卡：

- 先保体裁，再去味
- 正式、规范、克制、结构清楚本身不是 AI 味；保留合法的格式性结构和必要正式层级
- 先救长度和对象层，再处理句子顺滑度
- 不要把正式说明体系洗成另一种更顺滑的解释体系
- 事实类、政策、法律、医疗、技术、学术摘要和新闻 / 评论文本，不新增原文没有的事实、数据、身份、场景、时间地点、因果链或人物动机
- 正式讲话优先落回具体对象、场景、动作和使用情境，不要只剩“借这个机会讲三点”
- 短正式讲话若仍保留完整开场问候、三点展开、拔高意义、感谢收束四件套，默认未完成
- 短正式讲话如果还保留“今天不只是……更是……”“我想讲三点”“最后用一句话作结”这类发言模板骨架，也按未完成处理；去味不是把模板句换得更顺，而是把对象、用途、边界和后续动作留下来
- 讲话允许缩掉一层礼貌结构，优先保对象、用途、后续动作和边界提醒
- 短正式讲话不要把一个小设施、小空间、小启用仪式硬抬成“邻里互助 / 社区治理 / 公共精神”三段式讲话；对象越小，越要先落回谁来用、怎么用、边界在哪
- 政策评论要拆一处宏观稳句，换回具体地区、家庭压力、执行摩擦或判断分叉
- 政策评论若 marker 已降，但尾部仍是成熟政策评论员的总结块，也默认未完成
- 政策评论包里的首篇总论不要写成“完整主评论员定稿”；允许它只钉住一个主矛盾，不把托底分析、价值判断、治理导向一次讲圆
- 如果改后更像可直接上台或直接刊发的标准成稿，也要继续回写
- 如果政策分析把“经济 / 制度 / 伦理”之类分层讲得更顺、更齐、更完整了，但段群关系没被打散，也按未完成处理
- 如果原文本来是评论包中的一篇，改后却像独立完稿专栏，哪怕更顺也按未完成处理
- 混合评论件不要求“把三层都讲透”；一旦开始追求完整覆盖，通常就会重新长出标准政策评论壳
- 短评论 / 短议论文若开头已经把题目讲成“概念很好听 -> 现实没那么简单 -> 真问题在更深层”这类成熟专栏壳，改后仍一路稳稳讲透，也按未完成处理；至少要留一处现实摩擦、账目、操作成本或判断分叉，不要整篇像单口总论
- 编号执行摘录、条文说明、操作摘要如果被改得更像“完整制度解读 / 规范释义 / 可直接发布说明稿”，也按未完成处理
- 这类文本如果段题都反复写成 `关于……`，正文又靠 `并非 / 并不 / 不是……而是…… / 仍 / 已经` 这类整理词缝顺，默认仍在长正式壳
- 原文已经正式时，允许只减重复解释、空转限定和否定式归纳；不要求把它改得更顺、更会讲

### 诗歌 / 组诗

高风险信号：

- stanza、段群或意象推进被压扁
- 被改成更短的散文诗提纯版

执行卡：

- 优先恢复段群和留白
- 不要用更短、更亮的句子冒充“更像人写”

## 7. 混合体卡

适用：

- 长文里同时混着评论、叙事、方法、政策说明
- 同一文本里有多个运行方式在互相抢

先判主导运行方式：

- 是评论骨架在主导
- 还是叙事骨架在主导
- 还是说明体 / 政策体在主导

执行卡：

- 先救主导运行方式
- 不要把所有部分洗成同一种成熟解释者口气
- 如果混合体去味后只剩一种顺滑人格，也视为失败

### 设计外正式说明体

适用：

- 法规说明
- 摘录解读
- 执行口径文
- 工作通知 / 问答材料

高风险信号：

- 编号、条目、边界都保住了，但整篇仍像“稳定主持人”在控场
- 一条说明里连续出现限定、兜底、解释、免责四层壳
- 语言已经不是评论文，却还是明显像 AI 整理过的说明标准件
- 去味后只是更顺、更齐、更像规范说明件，说明壳本身并未拆掉

执行卡：

- 保住格式，不保住整套说明腔
- 能并回操作顺序、现场条件、责任动作的，优先并回，不让它们都站成解释句
- 连续三句都在“界定 -> 限定 -> 再解释”时，至少拆掉其中一层
- 如果只是把条文写得更顺、更全、更像手册，不算完成
- 如果文本本质上是“给执行者看的说明摘录”，允许把并列条目改成风险分组或步骤分组
- 不要让每一节都扩成同等密度、同等完整度的说明块；同密度本身就是 AI 壳
- 如果改后前言冒出完整通知口吻、礼貌抬头、背景交代和“现就如下”发布壳，优先压短或删去
- 元说明不能抢过操作说明；真正给执行者看的文本，应更快进入对象、边界、动作和例外
- 临时停水、停电、封闭、延时开放这类应急公告，不要被修成更完整的致歉型通知件；优先保时间、范围、原因、用户动作、恢复口径和联系方式，少做一层“请理解 / 感谢配合 / 尽量压缩影响”的安抚收束
- 原文已是正式执行摘录时，不要继续往“更像法规释义 / 更像制度解读 / 更像正式通知”方向升级
- 如果只是把词换得更法言法语、把句子配得更整、把禁止句写得更像模板法条，也算未完成
- 法规说明 / 办法摘录如果已经是编号条目，不要顺手把每一条都扩成“定义 -> 目的 -> 适用条件 -> 反面解释 -> 结尾导向”的完整说明块；执行者真正需要的是边界、动作、时点和责任链
- 工作通知 / 材料包说明如果同时把“为什么发、它不是什么、怎么理解、如何反馈、为何这样反馈、如何保密、为何保密、时间安排、常见问答”都讲成整齐长段，默认未完成；通知件优先保收件对象、版本、阅读顺序、提交动作、反馈口径和联系人
- 问答 / 通知材料里的答复不要每条都写成成熟培训讲稿；一问一答先解当前疑点，不必每答都回补背景、原则和安抚性总结
- 另一种回退是过度卡片化：为了去壳把知识库、内部说明、客服口径压成大量等长短句和单行条目，原本的上下文被删掉，剩下同一批关键词高频重复。此时不要继续压缩，改为并回相邻条件、动作、例外或短说明，让段落有功能差和长短差
- 评审意见回复 / 审稿回复不需要每条都写成完整小论文；保留 `意见 -> 回复` 的功能，但打散同一种 `已补充 / 已调整 / 本研究... / 后续研究...` 回答骨架，能说明具体改动、数据限制或不采纳原因即可，不必每条都补展望

### 访谈 / 口述整理

适用：

- 采访实录
- 口述记录
- 问答整理
- 现场转写稿

高风险信号：

- 仍保留问答格式，但答话被洗成成熟专访成稿
- 受访者每一答都像完整背景介绍、完整观点块或完整人生总结
- 采访人提问过于整齐，只负责把材料稳稳抬上去
- “第一次见到什么 / 为什么接手 / 最大困难 / 现在怎么看”这类答题卡骨架一格不漏

执行卡：

- 先保一问一答的真实功能差，不要把问答洗成正文段落
- 受访者的答话允许有绕、回看、补一句、停一下，不必每答都讲满
- 不要顺手把背景、动机、转折、意义一次补齐；一旦每个答案都像可直接刊发的人物专访块，默认未完成
- 采访人的问题不要全是稳稳抬题；允许保留一点追问、打断、转场生硬或信息未接上的地方
- 如果原文是口述整理，不要把受访者洗成“会表达的成熟叙事者”，更不要把整个访谈洗成杂志特稿的 Q&A 版

### 口播 / 社媒脚本

高风险信号：

- 原文是口播，改后却变成短标题分隔的使用说明、FAQ 或公告卡片
- 连续几段都在 `怎么做 -> 什么时候做 -> 不要做什么 -> 反馈给谁`
- 场景和对象还在，但说话流被切成等长操作块

执行卡：

- 保留口播的连续气口，不要把每个操作点都升成小标题
- 操作信息可以嵌在场景、提醒和具体动作里；同类禁止句合并，别连排
- 如果需要列点，保留少量口播转场和对象称呼，让它仍像说给人听，而不是贴在墙上的流程卡

## 8. 审计时要特别回扫

- `压缩 / 摘要化残留`
- `转场回潮残留`
- `安全限定回潮残留`
- `伪具体残留`
- `句法残留`
- `越界检查`

如果这些问题还在连续出现，就继续回写，不要停在“已经比原文自然了”。
