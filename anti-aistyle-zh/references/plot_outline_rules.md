# 剧情与提纲去 AI 味规则

这份文件处理：

- 长篇小说总纲
- 卷结构
- 章节规划
- 故事提要
- 人物弧线分配
- 多文件规划包

重点不是句子像不像 AI，而是结构方差够不够，块与块之间有没有真正异构。

## 快速路由

- 总纲、卷表、章节规划、人物弧线表、剧情摘要、执行表、多文件规划包才读本文件。
- 使用边界和多文件规划包先查第 0 节。
- 提纲壳、卷级结构和同模章节群查第 1-3 节。
- `hard fail` 的完整触发条件、固定动作和完成阈值只维护在第 4 节。
- 提纲+正文混合输入查第 5 节；输出和分批协议查第 6 节。
- 只做正文句子去味时不要套用本文件。

## 0. 使用边界

如果用户给的是总纲、卷功能、章节规划、人物弧线表、剧情摘要或执行表，但没有正文，也可以正常执行；此时直接修改已有结构文件，不等正文。

如果同时给了多份纯规划文件，先做 `规划包对表`，定四件事：

- 哪份是 authority file
- 哪些字段互相冲突
- 哪些 recurring system 属于书感母题
- 哪些只是被均摊到每卷的 quota

如果输入是超长纯提纲，先拆三层：

- `卷级链`
- `章节群链`
- `单章微链`

再抓：

- 显性脚手架
- `2-4` 组 `同模章节群`
- `前段 / 中段 / 后段 / 终局` 四段高风险重复区

## 1. 先拆的不是剧情，是壳

先把这些视为脚手架，不当成剧情本体：

- `卷主题 / 起点状态 / 终点状态 / 结构功能`
- `功能定位 / 主要场域 / 关键章节点 / 伏笔回收`
- `本卷功能 / 本章任务 / 节点回收 / 下一步推进`
- `类型与卖点 / 节奏补充 / 每50章一个高潮 / 热搜式名场面 / 阶段反派`

执行卡：

- 先拆脚手架，再决定故事怎么改
- 不只改结构分配，也要改提纲口气
- 去掉 planning shell，不让改后提纲继续像排产说明书
- 也不要把提纲去壳成 `故事底盘 / 主要人物 / 母题场域 / 分段章纲` 这类成熟故事 bible prose；规划文本要保工作感，不要改成成套世界观说明文
- 卷群文件优先写成 `这一卷群先做什么 / 这里卡在哪 / 争论点在哪 / 下一步被什么拖住` 的工作块，不要扩成顺滑的长段结构说明
- 不把 recurring system 一把删成空白，但也不要保留成“每卷都要均摊一次”的 quota

补一句硬约束：

- 总纲可以有总纲感，但卷群文件、关键章节群文件、终局安排文件不能一起滑成同一种“整书介绍口气”
- 关键章节群文件尤其不要写成“每章两三句解释 + 再补一层作者提示”的讲解稿；它该更像排险单、卡口单和取舍单，而不是章节导读
- 卷群结构文件也不要一开头就用两三大段 prose 先把这一卷讲圆；优先分成短块：入口、卡口、证据、分歧、收法
- 章节规划、关键章节群、终局安排文件里，不要频繁出现 `这一章不要 / 这章不是 / 真正重要的是 / 这里别再 / 更好的办法是` 这类作者讲解句；除非明确处于审计模式，否则它们会把规划文本写成讲义
- `12-24` 章级别的中短规划，不要先用一段 `全书抓手 / 前段怎么走 / 中段怎么走 / 后段怎么收` 把整份规划讲圆；总起最多留极短提要，主体仍应落在章块、章节群块和卡口块

## 2. 提纲层高风险卡

### 功能分配过匀

高风险信号：

- 每卷都能被一句话准确概括功能
- 每卷都同时推进主角、世界、关系、暗线
- 所有块都像稳定交付一组 KPI

执行卡：

- 允许某卷偏人物，不明显推进真相
- 允许某卷偏地方，不明显推进主角成长
- 允许某卷只压后效，不急着升级
- 粒度不要重新均匀化；有的块写卷级，有的块写章节群，有的块只保留一章

### 角色推进过于公平

高风险信号：

- 核心角色被平均轮到主场
- 每个人都准时拥有一卷自己的成长

执行卡：

- 允许某人连续两卷不亮
- 允许某人该爆发时只爆半次
- 允许某人先走错，再成长
- 不要把角色长期绑定成单一职能 lane

### 升级路径过于准点

高风险信号：

- 世界观一层层稳定抬高
- 每卷都比上一卷更大、更高、更深
- 几乎没有横移、停滞、白跑、误入

执行卡：

- 中段主动插入“不升级，只下挖”的卷
- 允许某卷看似支线，实际在改主线重心
- 大揭示后允许接一段明显变慢、变俗、变麻烦的后效

### 揭示机制重复

高风险信号：

- 连续多卷都靠旧物、残字、簿页、器物露底
- 连续多章都靠“懂行人认出 -> 规则露底 -> 章尾钉住”发亮

执行卡：

- 连续两次同类机制后，第三次必须换发亮方式
- 可换成人群误判、某人做错决定、关系断裂、地方秩序失衡、后效慢慢显出
- 如果前两章都靠 `露字 / 露簿 / 露签 / 器物指认` 发亮，第三章优先改成人群动作、空间结构、误认 / 错认或后效发亮

### 地方只在服务命题

高风险信号：

- 地方只是主题舞台
- 去掉地名后，故事功能几乎不变

执行卡：

- 让地理、习俗、节令、劳作、关系网反向逼剧情改变
- 让地方自己的麻烦拖慢或改写剧情

### 卷尾 / 章尾过于会收

高风险信号：

- 每卷准时钩下一卷
- 每章都在最后一击收住

执行卡：

- 主动保留一些“不漂亮但真实”的卷尾
- 允许某卷收在疲惫、误判、未消化的余震上
- 不要每卷都把命题重新钉牢

## 3. 同模章节群卡

至少做三件事：

- 抽 `2-4` 组 `同模章节群`
- 做至少 `1` 次互换测试
- 估一个主模板覆盖率

判断问题的简式标准：

- 互换地名、敌人、线索后仍然成立
- 前中后段只是换皮复用
- 读者已经能稳定预判“下一块怎么跑”

执行卡：

- 打散块长、粒度、句长和收口方式
- 插入白跑、误判、俗务、缓冲、后效
- 不要把旧模板改成另一种更规整的新模板
- 单块规划也不要滑成统一 `第X章《章名》 + 两三句说明` 的章节卡
- 关键章节群文件不要一章接一章都写成 `本章要点 + 作者解释 + 正确收法`；至少把一部分章块改成 `卡口 / 取舍 / 证据 / 后效` 视角，不要让每章都像导读卡
- 中短规划如果出现 `全书抓手 / 真正推动 / 最后分别落在……` 这种上帝视角总括块，默认继续压；这些话常把可执行规划洗成解释性 prose

## 4. Hard Fail 补充卡

这些情况直接按结构层 `hard fail` 处理：

- 卷长、章节群长度、节点密度对称得像排产
- `20+` 卷且卷长几乎全等
- 超过半数章节群共用同一条微骨架
- 前段还是单章细写，中后段却快速退成 `5-10` 章一包
- 最后 `20%-25%` 已进入统一终局坡道，且没有后效卷、误判段或余震收口
- 主模板覆盖率仍超过 `50%`
- 改后整份提纲开始像世界观说明、人物册、宣发故事 bible 或整书导览 prose，而不是案头结构文件

命中后固定动作：

- 不停在指出问题，必须继续改
- 默认把高风险卷 / 章节群直接改写出来
- 直到主模板覆盖率明显下降，异构块不再只是样板

完成阈值：

- 主模板覆盖率不再超过 `50%`
- 已至少打破 `3` 处统一卷长或统一章节群长度
- 终局不再是统一终局坡道

## 5. 提纲和正文同时存在

如果用户同时给了提纲和正文：

- 先判断问题落在哪一边
- 先找提纲和正文的偏差
- 先定这次是改提纲、改正文实现，还是两边都改

默认不要先把正文改顺去贴合提纲。
在这一步完成前，不进入句级润色。

## 6. 输出卡

如果输入主要是提纲，输出默认要包含：

- 改后的提纲正文 / 结构文本
- 被重构的高风险块
- 超长提纲分批时，每批至少落地一个高风险卷群或章节群
- 批首只标明本批范围，批尾最多一句续批提示
- 上下文不足以覆盖全书时，只处理已给材料，不虚构未给章节

不要只给：

- 修正原则
- 卷功能点评
- 每卷一句概括

如果改后出现下面这种新壳，也不算完成：

- 先用几段成熟 prose 讲世界、人物、母题
- 再用长段综述去包章节群
- 多文件包里的不同文件开始一起滑成同一种“整书介绍口气”
- 章节规划主体仍像作者在台上讲“这一章该怎么写、为什么这么写”，而不是案头真的能拿来排险和分配功能的结构文本

提纲层去味的完成标准，不是“指出结构太整”，而是“真的把结构打散并改出来”。
