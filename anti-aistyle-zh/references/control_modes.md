# 控制项与校准

这份文件只处理可选控制项和长文 / 多文件校准。
默认合同、统一入口、默认输出和 `hard fail` 以主 [SKILL.md](../SKILL.md) 为准，这里不重复展开。

## 快速路由

- 用户只给目标文本时，不需要读取本文件。
- 用户指定样文、改写强度、目标读感或输出模式时，先查第 1-5 节。
- 长文、多文件、分批输出或未知题材降级时，查第 6-7 节。
- 控制项只改变校准方式，不改变保真、事实边界和默认最终成稿合同。

## 1. 总规则

控制项全部可选；未指定时沿用主合同默认值。

控制项别名表：

| 用户侧名称 | 机器侧别名 | 默认值 |
| --- | --- | --- |
| `voice_sample / 样文` | `voice_sample` | 无 |
| `改写强度` | `rewrite_intensity` | `平衡` |
| `目标读感` | `target_reading_feel` | `更像自然中文` |
| `输出模式` | `output_mode` | `最终成稿` |

优先级固定为：

1. `voice_sample`
2. `目标读感`
3. `改写强度`
4. `输出模式`

如果 `voice_sample` 和 `目标读感` 冲突，不要静默吞掉下层控制项；在内部回执或诊断里写明：

- 这次优先保留了样文的什么
- 哪个目标读感被降级处理

## 2. voice_sample 卡

用户提供以下任一材料，就视为 `voice_sample`：

- 自己以前写过的样文
- 想接近的作者样文
- 同项目既有成稿

优先校准：

- 句长
- 段长
- 判断密度
- 转场习惯
- 收束力度
- 口语 / 书面比例

不要机械模仿表层词汇；优先保留推进方式、节奏、呼吸感和样文敢不敢停住。

## 3. 改写强度卡

默认选 `平衡`。

只有三种切换条件：

- 命中结构层 `hard fail`、长篇重构、用户明确要求更大胆改写：升到 `强力`
- 用户明确要求保留更多原样：降到 `保守`
- 其余都留在 `平衡`

### 保守

- 只动显眼解释句、总结句、骨架句
- 不主动重排段落
- 不主动重写剧情分配
- 不新增事实、背景、场景、心理或设定
- 不偷换叙述视角
- 但命中 `hard fail` 时也不能绕过结构重构，只能理解为“在必须重构的前提下尽量保真”

### 平衡

- 同时动姿态、段落和局部结构
- 允许压掉重复功能段
- 允许改段首领句和段尾收束
- 命中 `情节功能过顺` 或主导路线过整时，允许先列 `功能链 / 主导路线`，再做极小结构调整
- 默认不改叙事顺序，不改人物设定
- 输入主要是纯提纲 / 卷表 / 章节规划时，允许改卷功能、章节群分工、人物轮转、升级路径、揭示机制
- 命中 `hard fail` 时，也要继续做结构重构
- 默认不新增事实、背景、关系或安全氛围细节

### 强力

- 允许先抽结构，再回写正文
- 允许动章节群、剧情分配、揭示机制
- 允许重做一部分卷功能和收法
- 仍然禁止擅自新增关键设定、改名词体系、改立场

## 4. 目标读感卡

默认目标读感是：

- `更像自然中文`

如用户指定，可切到：

- `更像个人写作`
- `更像小说现场`
- `更像口语化非虚构`
- `更像克制评论文`
- `尽量保留原稿成熟度`

目标读感只影响：

- 收得多轻
- 讲得多满
- 留多少毛边

不影响保真底线；如果和 `voice_sample` 冲突，以样文为准。

## 5. 输出模式卡

未显式指定时，直接按主合同默认的 `最终成稿` 执行。

只有用户明确要求时，才切到：

### 句级标注

- 列出最值得动的句子或段落
- 每条附一句“为什么该改”

### 局部替代改写

- 只有用户显式指定时才使用；默认模式不输出多个版本
- 同一句给 `2-3` 个版本，短句替换可只输出替代句
- 每个版本前最多用短标签，如 `更稳：`、`更自然：`、`贴近样文：`
- 不写长诊断，不把替代句批量扩成报告
- 长文和提纲一般不建议使用；如用户指定，只选最关键片段替代，不把全文拆成碎片

最小约束：

- `更稳`：只做删解释、压总结、拆工整句、换抽象词
- `更自然`：可补一处原句已暗示的具体介质，但不新造背景、动机、关系
- `更贴近样文`：贴节奏、收束和判断密度，不借样文现成句
- 同一批输出里，不要反复用同一种救火模板句

### 分层审计

- 适合长文、长篇和提纲
- 先给层级判断，再决定是否回写

如果用户没有指定输出模式，不要反过来追问。

## 6. 长文与多文件边界卡

### 单篇短文

- `1500` 字以内：可直接改

### 中长文

- `1500-8000` 字：先看段落功能和长文气口
- 内部先形成主导路线；如果能轻易写成标准模板，再定 `2` 处骨架打断位，默认不前置展示

### 超长文 / 长篇原文 / 多文件

满足任一情况时，先抽结构：

- 超过 `8000` 字
- 超过 `3` 个文件
- 是整部长篇
- 是提纲 + 正文混合输入

先抽：

- 结构表
- 章节群
- 重复机制
- 角色轮转
- 卷尾 / 章尾收法

如果输入主要是超长纯提纲，再额外抽：

- 卷级链
- 章节群链
- 单章微链
- 脚手架拆除点
- `2-4` 组同模章节群
- `4` 个高风险重复区

进入回写前，内部可先给一个简短 `manifest`，只说明：

- 先处理哪一层
- 哪些文件或章节群一起看
- 哪些部分本轮暂不回写
- 本轮是否允许进入文本层

命中 `hard fail` 时，不要把高风险重复区挂进“暂不回写”。

分批输出协议：

- 每批仍以改后内容或改后结构为主
- 批首只标明本批范围
- 批尾最多一句提示下一批从哪里接
- 不把第一批写成总诊断、建议书或长 `manifest`
- 长篇原文先内部抽影子结构，再从用户给定批次或高风险章节群开始回写
- 超长提纲每批至少落地一个高风险卷群或章节群
- 上下文不足以覆盖全书时，只说明“本批按已给材料处理”，不虚构未给章节或文件

## 7. 未知题材降级卡

如果遇到当前没有专项卡的题材，不要硬套进最近的专项规则。

优雅降级顺序固定为：

1. 先按统一底层方法处理：只看解释、总结、骨架、句法、压缩回潮这些跨题材稳定问题
2. 如果文本明显带强体裁约束，但没有专项规则：进入 `保守边界模式`

保守边界模式：

- 优先保体裁
- 少动整体结构
- 只删明显空转解释、模板句和护栏句
- 不把文本洗成普通自然散文
- 不擅自补设定、补背景、改说话方式
- 不把说明体 / 规范体 / 纪要体 / 摘要体洗成另一种更顺滑的统一解释稿
- 不为了“更自然”补出新的提醒句、注意事项句、领路句或稳妥总结句

如果去味和体裁保真冲突，优先保体裁；仍拿不准时，退到局部替代改写或低风险去味。
