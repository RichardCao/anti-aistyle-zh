# 长篇提纲去 AI 味清单

这份清单是直接拿来审大纲、卷规划、章节表的。

不是改句子，是改规划感。

## 快速路由

- 纯提纲、卷表、章节规划或多文件规划包回扫时读本文件。
- 总纲、角色、升级、揭示、地方和卷尾问题查一至六节。
- 章节群、提纲壳和无功能场查七至九节。
- `hard fail` 完整条件、完成阈值和落地动作不在这里维护，统一回到 [plot_outline_rules.md](./plot_outline_rules.md)。
- 正文句子去味不要读取本文件。

## 一、总纲回扫

- 每卷是不是都过于明确地承担一组任务
- 每卷是不是都“既推进主角，又推进世界，又推进暗线”
- 是否存在某卷只承担后效、关系或地方纹理
- 是否有哪一卷故意不升级，只下挖
- 整书是不是能被直接列成一张任务表
- 是否先天带着一套会替剧情出声的脚手架字段
- 如果给的是执行表，`器物 / 节奏 / 章回 / 文风` 这些轴是不是也被均匀排产
- `跨卷回响 / 静戏缓冲 / 小人物回环` 是不是只写在表里，还没真正进入结构分配

## 二、角色回扫

- 核心角色是不是被平均轮到主场
- 是否有人物成长被拖延、误判、打断、走错
- 是否有角色长期只提供力量，不急着爆发
- 是否有角色的变化不是“变强”，而是“变窄 / 变冷 / 变重 / 变慢”
- 是否有人被长期稳定绑定成单一职能 lane

## 三、升级回扫

- 世界观是不是一层一层太准点地抬高
- 中段是否缺少横移、停滞、白跑和误入
- 是否所有大揭示都在预期位置发生
- 是否存在“以为升级了，实际只是生活更难”的段落

## 四、揭示机制回扫

- 是否连续多卷靠同一种东西露底
- 是否连续多章靠旧物、残字、簿页、器物承担最后一击
- 第三次同类揭示时，是否主动换了发亮方式
- 是否存在“真相不露底，只通过人心和秩序失衡显出来”的推进段

## 五、地方回扫

- 这个地域是否只在服务主题
- 去掉地名后，故事功能是否几乎不变
- 这个地方的劳作、节令、关系网是否反向塑形剧情
- 是否有地方自己的麻烦，逼得角色不能按计划推进

## 六、卷尾回扫

- 每卷是否都太会钩下一卷
- 是否有卷尾只是沉底，不重新拔高
- 是否存在“本卷的问题没说圆，但余震很真”的收法
- 是否总靠一个字、一个器物、一句钉子话来结卷
- 多份规划文件放在一起时，卷尾设计有没有互相打架，还没先做 authority 对表

## 七、章节群回扫

- 连续 5 到 10 章内，推进方式是否同质
- 读者能不能预判下一章又会翻出哪类旧底
- 是否存在高压连续太久、没有缓冲岛的区段
- 是否有看似低效但其实很养书的章节群
- 章节群长度是否对称得像排产
- 是否存在明显 `前细后疏`：前段单章细写，后段快速退成 `5-10` 章一包
- 章群内部是不是总在重复同一种五步微骨架
- 前中后段各抽一块，块级骨架是否只是换皮复用
- 任取两块互换地名、敌人、线索后，是否仍然成立
- 前段 / 中段 / 后段 / 终局是否都有异构块，而不是只局部摆样板
- 这一段里有没有连续高密使用同一种章尾刀法：
  - `字钉章尾`
  - `器物指认章尾`
  - `短问 / 短答章尾`
- 如果前两章都靠 `露字 / 露簿 / 露签 / 器物指认` 发亮，第三章有没有主动换成：
  - 人群动作发亮
  - 空间结构发亮
  - 误认 / 错认发亮
  - 后效发亮

## 八、提纲壳回扫

- 提纲正文是不是仍像 planner 在讲话，而不是像已经去壳后的结构文本
- 是否反复出现“本卷功能 / 本章任务 / 节点回收 / 下一步推进”这类壳话
- 脚手架字段是不是只是换了说法，又原样留在正文里
- 是否还保留 `类型与卖点 / 节奏补充 / 每50章一个高潮 / 热搜式名场面 / 阶段反派` 这类连载运营壳
- recurring system 有没有被误删成空白
- recurring system 是否又被保留成“每卷都要均摊一次”的 quota
- 改后是不是变成新的 `等宽章节卡 / 统一摘要卡 / 统一两句梗概体`
- 改后是不是滑成 `世界观说明 + 人物册 + 母题说明 + 分段章纲` 的成熟故事 bible prose
- 卷群结构文件是不是又被写成一段段顺滑说明，而不是工作块、分歧块、阻力块和高风险块
- 关键章节群文件是不是又在逐章讲“这章为什么重要、那章不要怎么写”，像作者讲解手册，而不是直接给结构落点
- 卷群文件开头是不是先用一整段 prose 把这卷“真正的意义”讲圆了；如果是，继续拆成短块
- 块长、句长、章名格式、收口句是不是反而比原稿更齐
- 章节群写法有没有从“旧模板”滑成“另一种更规整的新模板”
- 如果只是单卷或单块规划，改后是不是滑成统一的 `第X章《章名》 + 两三句说明` 小卡列队
- 单块规划去壳后，是否从“功能表口气”换成了“更文学、但同样整齐的章节命名卡”

## 九、无功能场回扫

- 是否有不推进主真相的小场
- 是否有俗务、误会、闲聊、白跑、等人、收东西、送东西这类低功率段
- 是否有人间麻烦反过来把大剧情拖慢
- 是否敢留“这一章没交付多少，但书更活了”的段落

## 十、最后一问

如果把大纲里的卷功能、角色任务、真相推进点、卷尾钩子全部列成表，它是不是过于整齐？

如果答案大体是“是”，就先不要急着补设定，先回到结构层判断；具体是否命中 `hard fail` 统一按 [plot_outline_rules.md](./plot_outline_rules.md) 第 4 节。

`hard fail` 触发条件、完成阈值和必须落地的结构动作，不在这份清单里重复展开，统一按：

- [plot_outline_rules.md](./plot_outline_rules.md)

这里只保留一句总原则：

- 命中结构层 `hard fail` 后，不要停在指出问题，必须继续做结构重构，并直接落地改后块。

先做这几件事：

- 先拆脚手架字段
- 拆掉一卷的标准功能
- 拖延一个角色的主场
- 换掉一组重复揭示机制
- 插入一段白跑或误判
- 把一卷的收法从钩子改成余震
- 打破一段对称卷长或对称章节群
- 推迟一个终局并线点
- 打散一组过于统一的章名、块长或摘要句长
- 如果只是单块规划，至少拆掉一处“整列章名卡”的写法，不要把旧壳换成更顺眼的新卡片
