# 小说长篇专项规则

这份文件专门处理已经修过一轮、黑词很少、但仍有残留 AI 味的长篇小说。它不是独立入口，也不是额外模式；主 skill 在判断文本是小说章节、连载正文或长篇主稿时，内部按需读取这里的规则。

重点不是禁词，而是：

- 跨章节重复的落锤模板
- 场景内句法过稳
- 作者替读者补半句
- 情节功能过顺
- 显性解释删掉后换成抒情翻译
- scene 自己成立前，解释动作先站出来
- 该保留的辨异句被误伤
- 视角进入点不足
- 多声部被修成同腔

核心风险之一，是作者解释动作打断 scene 自己成立，削弱沉浸感。

## 快速路由

- 小说章节、连载正文、长篇主稿、scene 群才读本文件。
- 只有长篇原文或提纲+正文混合输入，先查第 0-0.2 节。
- 单章和短 scene 优先查第 1-4.1 节。
- 跨章节重复、角色轮转和章尾同质查第 5-5.4 节。
- 不确定是否该调用时查第 7 节；非小说不要套用本文件。

## 0. 只有长篇原文时怎么做

如果用户给的是整部长篇原文，但没有总纲、卷表或章节规划文件：

- 仍然可以正常执行
- 但不要直接在剧情层全书盲改原文
- 先抽影子结构，再回写正文
- 详细顺序以主 `SKILL.md` 的决策树为准
- 分批处理时，每批仍直接输出改后正文；批首只标明章节 / scene 范围，批尾最多一句续批提示，不把第一批写成总诊断
- 如果上下文不足以覆盖全书，只按已给材料抽影子结构，不虚构未给章节

影子结构至少包括：

- 隐式卷功能
- 章节群边界
- 角色主场轮转
- 升级路径
- 揭示机制分布
- 章尾 / 卷尾收法
- 卷气口卡
- 角色阶段卡

如果只是单章或少量摘录，不必先抽整书结构，可直接按本文件规则改局部。

但只要单章已经命中：

- `情节功能过顺`
- `线索 / 证人 / 反转来得太准`

也先列一行 scene `功能链`，再决定是否能直接改句子。

## 0.1 提纲和正文同时存在时怎么做

如果用户同时给了：

- 总纲 / 卷功能 / 章节规划
- 多章正文 / 长篇主稿

先做 `提纲-正文偏差诊断`，先判断：

- 是提纲太整
- 还是正文把原本可用的结构写顺了
- 还是两边都要改

在这一步完成前：

- 不要先修句子
- 不要先把正文改顺去迎合提纲
- 不要假装所有问题都能靠局部 scene 改掉

## 0.2 长篇结构闸门

只要命中主 `SKILL.md` 的长篇结构闸门，就先停在结构层。

在还没列出这些项目之前，句级动作只能举证，不能回写：

- 隐式卷功能
- 章节群边界
- 重复模板
- 角色主场轮转
- 揭示机制
- 高风险章节群
- 章尾 / 卷尾收法
- 卷气口卡
- 角色阶段卡

先补三张短卡：

- `卷气口卡`
  - 温度
  - 速度
  - 空间手感
  - 情绪主轴
  - 最怕同色
- `章尾簇`
  - 最近一段高密章尾类型
  - 哪些章尾该保留
  - 哪些章尾必须拆异
- `角色阶段卡`
  - 骨相不变项
  - 当前阶段变化
  - 最容易滑成作者手感的微反应模板

## 1. 小说里的对比句豁免

不要机械打掉“不是……而是……”。

先判断它在干什么。

### 应优先保留的情况

- 在做辨异：把“看错”改成“看清”
- 在做识险：指出异象真正落点
- 在做动作判断：说明主角为何改动作
- 在做认知推进：从假线索转到真线索

这类句子常见于：

- 志怪
- 悬疑
- 玄幻
- 侦查型场景

如果句子承担的是：

- 揭异
- 破障
- 认门
- 认灯
- 认名
- 认路

默认先保留。

### 应优先处理的情况

- 作者替读者归纳意义
- 情绪已经成立后再补一句“真正难的是……”
- 现场已经够了，还要替文本下定义

## 2. 情绪落锤模板簇

长篇小说里，真正容易留下 AI 味的，常常不是一个词，而是一组反复承担相同功能的模板。

高风险模板：

- 第一次真正……
- 终于真正……
- 这时才真正……
- 这才真正……
- 这回是真……

问题不在单句，而在：

- 多个章节都用同一把锤子敲情绪节点
- 角色的“终于成立”被同一种词头反复落印

处理方向：

- 不逐句硬删
- 先统计是否跨章节重复承担同功能
- 如果重复明显，优先改其中一半以上
- 改法不是换同义词，而是让情绪落在动作、迟疑、停顿、呼吸、物件反应上

## 3. Scene 内三连稳句

高风险表现：

- 连续 `3` 句以上都很完整、很稳、很会收
- 常见结构是：
  动作成立 -> 精准比喻 / 状态判断 -> 再补一句稳妥解释

单句往往不差，连起来就容易像“精修腔”。

处理方向：

- 拆掉其中一句解释
- 让一处停在动作
- 让一处停在物件
- 让一处只留声音或身体反应

不要把整个 scene 都修成松散口语，只拆过稳的那一节。

## 3.1 视角进入点

高风险表现：

- scene 一开头就只剩交代，没有进入口
- 人物已经在场，但读者摸不到他先看到、先听到、先误认到什么，也摸不到代词、时空或注意力的落点
- 改后信息更清楚了，scene 却更像俯视说明，不像从人里进去

处理方向：

- 每个关键 scene 至少保一个进入点：
  - 代词和时空锚点
  - 先入眼的物
  - 先撞到耳朵的声
  - 先卡住身体的阻力
  - 先冒出来的误认或迟疑
- 不要把进入点全换成“作者替读者整理现场”
- 如果现场只剩信息和动作，就补一个视角锚点，不要补解释句

## 3.2 任务 scene / 小队分工模板

高风险表现：

- 一到站、一进屋、一开门就开始按名单分工
- `谁进值班室 / 谁守门 / 谁补图 / 谁留车上` 这种 roster 一列出来，scene 立刻滑成程序推进
- scene 的推进主要靠“按部就班找到下一件物 / 下一张单 / 下一站提示”
- 线索物件摆得太正，像专门等人来取

处理方向：

- 不要让多人任务 scene 靠完整分工表来获得清楚感
- 至少打断一处：分工不全说、先出小阻力、先露误判、先让某人身体或注意力卡一下
- 线索不要一拿就能直接接上下一步；允许少半步、错半步、先只知道物件，不立刻知道用途
- 如果 scene 已经有了环境压力和人物动作，就不要再补一整套程序说明把它讲顺
- 一旦出现 `到站 / 进楼 / 开门`，不要立刻接 `谁去哪里 / 谁守哪里 / 谁带什么 / 下一站在哪` 的整齐排班；宁可少讲一项，也不要把 scene 交还给任务表
- 便签、权限单、记录簿、工程图、广播残句这类线索，不要连续都做成高可读、高可执行的提示卡；时间、地点、人物、柜号、路径不要每次都交代齐
- 特别要防“提示包叠提示包”：一张纸告诉柜号，柜里再给图，图上再给时间，下一处再给外貌识别和路线，这种层层接力即使句子自然，也还是程序壳
- 发现线索时，优先让人物先处理“真假 / 缺漏 / 看不懂 / 用不上”中的一个，不要每次都顺利转成下一步行动

## 4. 作者半句翻译

这类是小说里最该审的残留。
判断标准不只是像不像解释，而是它有没有抢走现场本来可以自己完成的沉浸。

高风险表现：

- 人物状态已经立住，作者再补一句解释
- 情绪已经到了，作者再替读者翻译其分量
- 气氛已经成了，作者再补一句“这意味着……”

常见句型：

- 声音仍冷，却比先前更实
- 像人在真正被旧史砸中以后，反倒会先把声收回来
- 他这时才真正……

处理方向：

- 能删就删
- 能并进动作前后就并
- 能改成一个物件反应就不留解释句

## 4.1 伪去味

高风险表现：

- 明面上的解释句删掉了
- 但换成了更软、更抒情的翻译句
- 读起来不像解释腔了，却还是作者在替读者收情绪

常见信号：

- 潮气、凉意、声气、灰尘、回声被拿来承担“余味盖章”
- scene 已经站住，段尾还想再留一个设计过的意味句
- 短 scene 删掉解释后，被收成更干净的“文学短页”：一个旧物、一个停顿、一个短硬尾句在替作者兜底
- 结尾只剩某个物件或动作的回声，读起来像特意为读者留的一刀，而不是 scene 自己停住

处理方向：

- 不要只把解释句换成氛围句
- 能停在动作、停顿、物件或未完成对话，就先停住
- 物件揭示后，不要顺手补一整段“这很像她 / 他这一辈子向来如此 / 重要的东西总藏在琐事里”式人物总括
- 尤其要防 `线索 / 物件出现 -> 人物反应 -> 立刻理解 -> 顺手给人物总括` 这一整条链；打断其中至少一环，不要让小 scene 自己把意义讲完
- 遗物、钥匙、便签、旧衣、保温杯这类触发器最容易把 scene 拉回“像她 / 没说完的话都在这里 / 剩下要他自己去开”式收束；默认只留反应、动作、外部声响和没处理完的事，不补象征解释
- 触发器出现后，如果人物已经知道“接下来要去哪里 / 要做什么”，优先停在这个现实动作，不再补一层“这意味着什么”的尾句
- 也不要借便签、遗物去回补“她向来就是这样的人”这一层人物通性；scene 里能知道的是这张纸写了什么、此刻人怎么动，不是顺手把整个人讲明白
- 更硬一点说：看到便签、钥匙、旧衣以后，禁止再写 `这确实像她 / 她向来如此 / 重要的东西总藏在……` 这类判断句，哪怕只剩一句也算回退
- 如果原稿只有一小段解释，改后却扩成更长的心理理解段、关系总结段或人物通性段，也算伪去味
- 短 scene 不要为了“更像小说”再补一枚设计过的尾句；如果现场已经够了，宁可停在未完成动作、未接上的话或人还没收回去的姿势
- 也不要把短 scene 剥到只剩 `几句对话 + 一个动作 + 一声响 / 一件物` 的精修小页；至少留一点不为情绪服务的低功率杂质，比如门框的灰、电视里的天气、桌边没动的汤、手上没擦干净的汁水
- 另一类伪去味是把原来的硬解释改成“更像电影”的物件调度：椅子像刚坐过、门像刚翻过、缝像在等人钻进去、声音像有人正好从前头拖过去。若这类“方便推进的可疑痕迹”连续出现，也是在替 scene 铺路
- 旧人重逢 / 巷口短见面 / 药店门口这类小 scene，不要顺手收成“成年人终于看清彼此位置”的完整感伤页；人物说到够用就停，别再补一层时间把所有人推回各自位置上的成熟总结
- 这类短见面尤其要防“递纸巾 / 留下一件小物 / 折好塞进口袋”被写成整场相遇的盖章尾物；物件可以留，但不要替 scene 完成情绪收束
- 还要防“过于体贴的悬疑物证”：像 `刚被放下不久`、`封面反而很干净`、`几页纸被人夹好`、`左脸靠耳朵有疤`、`今晚穿深灰风衣` 这种专门帮人物省掉辨认和筛选成本的写法，一旦连着出现，就是线索便利化，不是现场感
- 判断标准仍然是：scene 能不能自己立住，而不是作者再补一层余味
- 审计时单独检查“是不是只是换了一种更软的翻译”

## 5. 跨章节模板重复

有些问题单章看不出来，长篇才看得见。

高风险表现：

- 多章反复用同一种情绪落点
- 多章反复用同一种视角矫正句
- 多章反复用同一种“像……却……”句法
- 多章反复在关键节点让人物“第一次真正……”一次

处理方向：

- 不在单章内处理完就算
- 要做章节级抽样复核
- 优先改那些跨卷重复、功能相同、位置相似的句子

## 5.0 多声部与对白区分

高风险表现：

- 主要人物说话都像同一个作者在换名字发声
- 年龄、身份、地位、熟疏关系不同，但对白的节奏、词性、收尾习惯很像
- 地域、阶层、职业、代际差异原本存在，改后却被洗成统一标准语
- 对白被修得更整后，信息更清楚了，人物却更难区分

处理方向：

- 回写前先看主要人物是不是各有一套稳定但不模板化的说话重心
- 区分人物时，不靠口头禅堆标签，先看句长、停顿、回避方式、顶撞方式和信息暴露习惯
- 不要把旁白、对白、引语都修成同一种成熟叙述气口
- 如果两个人的对白互换后也基本成立，就继续改

## 5.1 功能链过顺

高风险表现：

- 旧物、残字、账页、证人、转折像在按顺序补位
- 线索一到，解释立刻就到
- 证人和物件都太会出现在最合适的位置

处理方向：

- 先列出 `线索 -> 证人 -> 物件 -> 认知推进` 的功能链
- `平衡` 档也允许极小结构调整：
  - 延迟一个证人
  - 让一个线索先不可用
  - 让一个物件只给半条信息
- 先打断“环环到位”，再做句级去味
- 短章节如果已经出现 `看见物件 -> 想起 / 理解 -> 给人物下定义`，优先拆后两步；宁可只留下反应和误差，也不要补成完整认知推进
- 打断之后再检查后半程是否重新汇流成 `物证 -> 人证 -> 公共见证`
- 对“证人迟到但准点到齐”“雨夜补件后次日公开”这类成熟收法要单独留意
- 如果线索链已经把 `时间戳 / 柜号 / 地图 / 口令 / 外貌特征 / 赴约时间` 连成一条可执行路线，默认判为过顺；同一组线索里出现四项以上，也按“说明书式推进”处理
- 能删掉一环最好；删不掉时，至少让其中一环变模糊、变滞后、变成误导，别让主角像按说明书推进
- 如果人物到现场后几乎不需要筛选、试错、比对、犹豫，就能从物件一路走到人证或下一地点，也默认判为过顺；scene 需要一点搜索成本，不要全替人物省掉
- 角色就算自己说“线索太完整 / 太巧 / 太像安排好的”，也不算免责；如果行动上仍无缝接收、无缝转场，问题还在
- 旧案 / 追查章如果一路按 `新物证 -> 老证人 -> 老照片 / 老账册 -> 指向旧人 -> 当章供述` 顺排到底，也按过顺处理；宁可留一处断口，也不要在一章里把旧案查成完整答案
- 旧案章还要防另一种收法：`真相落地 -> 遗骸 / 物证补齐 -> 案卷定论 -> 桥边 / 雨夜 / 敬亡余韵` 一口气收完。即便前面查到了，也不要在同章把法律、情感和象征性告慰全部收圆

## 5.2 相邻章节发亮机制轮换

高风险表现：

- 相邻几章都靠同一种东西发亮
- 露字、露簿、露签、器物指认连续高密出现
- 地域、人物和冲突换了，scene 点亮方式却没换

处理方向：

- 如果前两章已靠同一种发亮机制成立，第三章必须主动换
- 可轮换为：
  - 器物发亮
  - 人群动作发亮
  - 空间结构发亮
  - 误认 / 错认发亮
  - 后效发亮
- 不要让一整段章节都像“换皮后的同一种点灯手法”

## 5.3 章尾同质簇

高风险表现：

- 一长段章节都在用同一种利收
- 常见高密章尾包括：
  - `字钉章尾`
  - `器物指认章尾`
  - `短问 / 短答章尾`

处理方向：

- 不只记录 `章尾 / 卷尾收法`
- 还要按最近一段章节做一次 `章尾簇` 抽样
- 如果同一类型高密连用，优先把其中一半以上改向：
  - 人的动作
  - 地方余震
  - 未说完的人话
  - 局部沉底

## 5.4 卷气口与角色阶段

高风险表现：

- 卷与卷之间只有事件差别，没有呼吸差别
- 中段连续几卷同温、同速、同一只手在控场
- 主要角色后期一起滑成统一的沉静判断句

处理方向：

- 每卷先定一张最短 `卷气口卡`
- 每个主要角色先定一张最短 `角色阶段卡`
- 回写时先保护：
  - 这卷的温度 / 速度 / 空间手感
  - 这个角色此阶段的骨相和变化
- 不要把四个主要人物都修成同一种熟手微反应模板

## 6. 哪些东西不要乱动

小说里这些东西即使重复一点，也未必是 AI 味：

- 地方性器物
- 志怪世界里的辨异句
- 角色口头禅
- 世界观核心母题的有意回环
- 收束但不解释的短硬句

判断标准：

- 它是不是这个世界自己的说法
- 它是不是这个角色自己的手感
- 它是不是在推进现场，而不是整理答案

## 7. 什么时候调用这份文件

这些情况优先读这份文件：

- 用户给的是小说章节或长篇主稿
- 用户只给了长篇原文，没有提纲文件
- 文本已经修过一轮，显性黑词不多
