# anti-aistyle-zh

`anti-aistyle-zh` 是一个面向中文写作场景的 Codex skill，用来降低中文文本里的 AI 味、模型腔和模板感，同时尽量保留原意、事实、体裁、口吻和使用边界。

它适合处理那些读起来过度解释、过度总结、太顺、太完整、太像汇报稿 / 课件稿 / 模型输出的中文文本。

## 能做什么

- 把中文文本改得更自然、更接近具体作者或具体体裁的读感。
- 保留原文的意思、立场、事实、体裁和发话人声音。
- 用一个统一入口处理短段、文章、评论、随笔、小说 scene、长文、总纲、卷规划和章节规划。
- 不只替换词语，也检查解释姿态、段落结构、句法节奏、过度工整的转场、过度总结和体裁漂移。
- 对强体裁保持谨慎，包括通知、FAQ、知识库、客服回复、口播、社媒脚本、审稿回复、政策材料、小说和剧情提纲。

## 不是什么

- 不是检测器绕过工具。
- 不承诺降低检测器分数。
- 不通过错别字、噪音、无支撑细节、假停顿、随机口语化来伪装“人味”。
- 不把所有文本都改成碎口语。

## 默认行为

默认只需要提供目标中文文本。

默认设置：

- 改写强度：`平衡`
- 目标读感：`更像自然中文`
- 输出模式：`最终成稿`

诊断、manifest、审计和执行回执默认都是内部过程。除非用户明确要求分析、审计、验证或回归式输出，否则 skill 优先返回改后的中文成稿。

## 适用场景

当用户提出这些需求时适合使用：

- `去 AI 味`
- `AI 味重`
- `模型腔`
- `ChatGPT 腔`
- `太像 AI 写的`
- `减少机器生成痕迹`
- `解释腔`
- `总结腔`
- `汇报腔`
- `课件腔`
- `翻译腔`
- 中文文本太顺、太完整、太结构化、太模板化

如果用户只是要求“更正式”“更营销”“更像公文”，不要仅凭这一点触发本 skill。
